# Whitepapers

The three core Dytallix whitepapers are bundled directly in this repository so
they can be referenced from the docs site and the GitHub repo without relying on
external file hosting.

Use the whitepapers for long-form design context. For integration details, use
the rest of this docs set and the current public gateway behavior.

## Recommended Reading Order

1. foundational
2. technical
3. tokenomics

## Important Caveat

The whitepapers describe the intended protocol design. The live public testnet
can move faster or slower than those documents in individual areas.

Current example:

- some tokenomics language describes `DRT` as central to fee economics
- the live public node currently reports `udgt` as the fee denomination

For builders, use the whitepapers for context and the current API docs for
implementation behavior.
